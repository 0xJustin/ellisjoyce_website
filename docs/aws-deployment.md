# AWS Deployment Runbook

## Recommended architecture
- `S3` private bucket for static files
- `CloudFront` distribution in front of S3 for HTTPS + global caching
- `Route 53` (or external DNS) pointing `www.ellis-joyce.com` and apex to CloudFront
- `ACM` certificate in `us-east-1` for your domain

## Deploy steps
1. Build final site bundle and verify route coverage:
   - `npm run prepareDist`
   - Output artifact: `dist/`
2. Create S3 bucket and upload:
   - `npm run deployS3 -- <your-bucket-name>`
   - Optional full one-command release (build + upload):
     - `npm run releaseS3 -- <your-bucket-name> [distribution-id] [base-url]`
   - or `aws s3 sync dist/ s3://<your-bucket-name>/ --delete`
3. Create CloudFront distribution with origin access control.
4. Configure custom error response:
   - `404` -> `/404.html`
5. Attach ACM certificate and set alternate domain names:
   - `ellis-joyce.com`
   - `www.ellis-joyce.com`
6. Update DNS records to CloudFront target.
7. Invalidate CloudFront cache after major updates:
   - `aws cloudfront create-invalidation --distribution-id <id> --paths '/*'`
   - or `npm run deployS3 -- <your-bucket-name> <distribution-id>`
8. Run smoke tests after DNS propagation:
   - `npm run smoke -- https://www.ellis-joyce.com`

## Build behavior
- New actively-managed pages are rendered from `src/pages/`.
- Legacy sentimental snapshots are copied into `public/` by `scripts/import_legacy_pages.sh`.
- Featured photography assets are synced into `public/assets/` by `scripts/copy_featured_photos.sh`.
- Sitemap is generated by Astro (`@astrojs/sitemap`) into `dist/sitemap-index.xml`.

## Caching behavior in deploy script
- HTML and metadata files are uploaded with:
  - `Cache-Control: public,max-age=0,must-revalidate`
- Static assets under `/assets/` are uploaded with:
  - `Cache-Control: public,max-age=31536000,immutable`

## Redirect behavior
Legacy compatibility pages are static HTML redirects for:
- `/home` -> `/`
- `/cart` -> `/photography`
- `/search` -> `/blog`

If you want edge-level redirects, use CloudFront Functions after initial cutover.
