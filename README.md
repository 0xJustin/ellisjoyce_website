# Ellis-Joyce Self-Hosted Website

This repository contains a self-hosted static replacement for `www.ellis-joyce.com` with two priorities:
- preserve legacy sentimental pages at original URLs
- establish a clean, extensible base for CV/blog/photography growth

## Project layout
- `src/pages/`: Astro-managed pages (new and actively developed routes)
- `src/data/birds/`: generated ABA/eBird catalog plus manually maintained seen/photo metadata
- `public/`: static and legacy URL snapshots copied directly into build output
- `dist/`: deployment artifact generated by Astro
- `archive/`: backup snapshots and mirrored assets from legacy Squarespace site
- `migration/`: URL discovery and manifest files
- `scripts/`: import/build/verification helpers
- `docs/`: deployment and migration notes

## Quick start
1. Install dependencies:
   - `npm install`
2. Build a deployment-ready artifact:
   - `npm run prepareDist`
3. Start local development server:
   - `npm run dev`
4. Optional route verification against migration manifest:
   - `npm run verifyRoutes`
5. Optional local HTTP smoke test of all discovered routes:
   - `npm run smokeLocal`

## Storyblok (optional CMS)
- Storyblok support is wired for homepage, photography, and new blog posts under `blog/*`.
- Create `.env` from `.env.example` and set `STORYBLOK_DELIVERY_API_TOKEN`.
- Full setup steps: `docs/storyblok-setup.md`

## Important routes
- New hub: `/`
- CV pages: `/justin`, `/gwen`
- New blog index: `/blog`
- Interactive demo: `/blog/interactive-neuroscience-demo`
- Rewilding map prototype: `/blog/rewilding-map-prototype`
- New photography hub: `/photography`
- Bird atlas database: `/photography/birds`
- Legacy wedding route examples: `/weddinghome`, `/details`, `/registry`
- Legacy blog route examples: `/ellis-joyce/*`, `/ellisjoycemonthly/*`

## Bird atlas workflow
1. Rebuild the taxonomy catalog (ABA + eBird phylogeny):
   - `scripts/build_bird_catalog.py`
2. Mark seen species manually:
   - edit `src/data/birds/personalStatus.ts`
   - keys use eBird `speciesCode` when available (fallback is ABA alpha code)
3. Add your photos:
   - place files under `public/assets/images/birds/`
   - register each photo record in `src/data/birds/personalPhotos.ts`
4. Build and preview:
   - `npm run build`
   - `npm run preview`

## Build and output
- `npm run build` writes static output to `dist/`
- `npm run prepareDist` synchronizes legacy pages and photos into `public/`, builds Astro output, validates route coverage, and runs local smoke tests
- `npm run preview` serves the generated `dist/` build
- `npm run deployS3 -- <bucket-name> [distribution-id] [base-url]` uploads `dist/` to S3, optionally invalidates CloudFront, and optionally runs post-deploy smoke tests
- `npm run releaseS3 -- <bucket-name> [distribution-id] [base-url]` runs full release (`prepareDist` + `deployS3`)
